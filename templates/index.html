About APP

Key features of the assignment

  • Calculator

    Within the Calculator section, you will find four different modals, one for each operator which either gets a local result using regular JS or uses a request method to fetch one from the Flask server. Furthermore, each section has been fitted with "not so over the top stylisation", this is to help keep each section simpler to ensure anyone can understand. Lastly, I've fitted the division operator with a logical statement, this is to ensure when the divisor is equal to 0 or empty, it will return "#DIV/0" rather than infinity which is incorrect.

  • Journal

    Within the Journal section, you will be able to find logs of each week up until this project was completed. All journal entries are stored in a JSON which is then hosted on the Flask server, so that the JS can populate the Entries field. On top of that, users can "add" and "delete" without pushing each entry individually. Instead they can use the "Upload Journals" button and publish the updated list which will retain on reload. Lastly, when entering a custom value within the "ID" section it will append that value with value entered to the current JSON, however, if the value is left blank or 0, it will append to the next index that isn't custom fitted.

  • Design

    The design approach I have taken to create the theme of this website was highly inspired by "Discord.Com's" website using two main colours only, for example, several shades of purple and different shades of white. On top of that, when designing the website I had thought of a simplistic way to give users a fresh look, whilst looking like the typical one-page-websites to ensure that consistency is kept. Lastly, the responsiveness. I have taken the flex route instead of the boot- strap framework, this was so I could get a deeper understanding of responsiveness and format, instead of having a pre-text that'll do it all for me.

  • Thoughts

    The Thoughts section. This section as seen above is a way to engage users when viewing the website, much like the Journal section, each thought can be added or deleted by selecting the Admin list button. However, a thought can not be chosen, instead they are given at random whether you are refreshing or on first load.
